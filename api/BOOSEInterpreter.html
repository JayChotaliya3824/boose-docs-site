Table of Contents

Namespace BOOSEInterpreter

Classes

ArrayCommand

The ArrayCommand class is implemented to handle the creation and initialization of arrays. Support is provided for both integer and real number arrays, including single and multi-dimensional definitions.

BOOSEException

The BOOSEException class is implemented to handle application-specific errors within the BOOSE Interpreter. It is derived from the standard Exception class to provide custom error messages and handling.

BaseCommand

The BaseCommand abstract class is implemented to provide shared functionality across specific command implementations. A common expression evaluator and helper methods are exposed to derived classes to facilitate argument processing.

BooleanCommand

The BooleanCommand class is implemented to facilitate the creation and assignment of boolean variables. Logical expressions are evaluated, and the resulting truth values are stored within the application state for later use.

CallCommand

The CallCommand class is implemented to facilitate the invocation of user-defined methods. Parameters are parsed, evaluated, and passed to the method body for execution within a local variable scope.

CastCommand

The CastCommand class is implemented to handle variable type casting. Real number values are converted into integers and stored in destination variables.

CircleCommand

The CircleCommand class is implemented to facilitate the drawing of circles on the canvas. The radius expression is evaluated, and the corresponding drawing operation is invoked.

CommandFactory

The CommandFactory class is implemented to manage the creation and retrieval of command objects. The Factory design pattern is utilized to instantiate commands based on string identifiers, ensuring a decoupled architecture. The Singleton pattern is applied to ensure a single instance of the factory manages command generation throughout the application lifecycle.

CommandHelper

The CommandHelper class is implemented to provide utility methods for command execution. Common operations such as argument evaluation are centralized to ensure consistency and reduce code duplication.

ControlFlow

The ControlFlow abstract class is defined to serve as a base for all flow control structures. The starting line index of the control block is stored.

DrawToCommand

The DrawToCommand class is implemented to handle drawing lines to specific coordinates. The pen is moved from its current position to the specified target coordinates, creating a line on the canvas.

DrawingCanvas

The DrawingCanvas class is used to perform drawing operations on a bitmap surface. It extends the functionality of the BOOSE.Canvas class to support specific shapes and text rendering using GDI+.

ExpressionEvaluator

The ExpressionEvaluator class is implemented to process and evaluate mathematical and logical expressions. A DataTable is utilized for numerical computation, and custom logic is applied for string operations and boolean comparisons.

FillCommand

The FillCommand class is implemented to control the fill state of shapes drawn on the canvas. The fill status is toggled based on the provided argument, determining whether subsequent shapes are filled or outlined.

ForControl

The ForControl class is implemented to manage the state of 'for' loops. The loop variable, start, end, and step values are maintained.

Form1
IfControl

The IfControl class is implemented to manage the state of 'if-else' blocks. The condition and the execution status of the branch are tracked.

IntCommand

The IntCommand class is implemented to facilitate the declaration and assignment of integer variables. Functionality is provided to parse variable declaration syntax and evaluate initial values if supplied.

MethodDefinition

The MethodDefinition class is implemented to store the structure of user-defined methods. Information regarding parameters and the sequence of method body commands is maintained for execution.

MoveToCommand

The MoveToCommand class is implemented to handle the movement of the drawing pen. The pen position is updated to the specified coordinates without performing any drawing operation on the canvas.

Parser

The Parser class is implemented to process and execute BOOSE scripts. Script commands are analyzed line by line, flow control statements are managed, and methods are defined and invoked. The parser interacts with the CommandFactory to instantiate commands and the DrawingCanvas to reflect visual outputs.

PeekCommand

The PeekCommand class is implemented to retrieve values from arrays and store them into variables. Support is provided for both one-dimensional and two-dimensional arrays of integer or real types.

PenColourCommand

The PenColourCommand class is implemented to update the pen colour used for drawing operations on the canvas. Colours are defined using RGB values provided as parameters.

PokeCommand

The PokeCommand class is implemented to assign values to specific elements within arrays. It supports updating values in both one-dimensional and two-dimensional arrays of integer or real types.

RealCommand

The RealCommand class is implemented to handle the declaration and assignment of real (floating-point) variables. It provides functionality to parse variable declaration syntax and evaluate initial values if they are supplied.

RectangleCommand

The RectangleCommand class is implemented to facilitate the drawing of rectangles on the canvas. It processes width and height arguments, evaluating expressions if provided, and invokes the rectangle drawing method on the canvas.

VarCommand

The VarCommand class is implemented to handle the assignment of values to existing variables. It parses the assignment expression, evaluates the result, and updates the variable's value in the shared state.

WhileControl

The WhileControl class is implemented to manage the state of 'while' loops. The loop condition string is stored for re-evaluation.

WriteCommand

The WriteCommand class is implemented to display text or values on the drawing canvas. It processes arguments which may include text strings, variables, or expressions, and renders the result at the current pen position.

Interfaces

ICommand

The ICommand interface is defined to establish a contract for all command implementations. A standardized Execute method is required, ensuring consistent execution across different command types within the interpreter.
